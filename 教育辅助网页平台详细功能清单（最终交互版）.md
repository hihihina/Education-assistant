# 教育辅助网页平台详细功能清单（最终交互版）

## 核心运行原则（100% 严格执行，无突破）

1. **无角色区分**：网页无任何用户角色划分，无强制注册登录要求，所有访问用户均可无差别使用全部功能，无任何权限限制
2. **零持久化存储**：所有用户上传的 Excel 数据、输入的文本内容、API 配置信息、分析结果、对话记录，均仅临时缓存于当前浏览器会话中；页面关闭、刷新或会话超时后，所有数据自动彻底清除，不进行任何本地 / 云端的持久化存储
3. **单次数据绝对聚焦**：所有成绩分析能力 100% 基于用户**当前单次上传的 Excel 数据集**，不关联任何过往数据、不做跨数据集对比、无任何历史数据累计相关功能
4. **AI 纯文本限定**：AI 问答功能仅支持纯文本输入，不提供任何图片上传、识别、解析相关能力
5. **匹配规则唯一**：Excel 数据提取仅采用分层权重关键词匹配模式，无任何兜底校准、智能映射功能，匹配失败直接返回结果，无额外处理流程
6. **网页全屏可视化**：成绩分析结果页采用全屏沉浸式布局，数据、表格、交互式图表（非图片）占满整个网页视口，无冗余导航干扰

## 一、单次成绩分析功能

### 1.1 Excel 数据导入与分层权重关键词匹配预处理

#### 1.1.1 基础导入与格式前置标准化

-  支持.xlsx/.xls 格式 Excel 文件单次拖拽上传或点击选择上传，兼容主流 Excel 版本，仅处理单 Sheet 数据，默认选中数据量最大的有效 Sheet

-  全程前端本地解析，文件读取、格式处理、数据提取均在用户浏览器完成，仅匹配成功的有效数据用于后续分析

-  前置格式标准化处理（仅为关键词匹配提供合规表头，无兜底修正）：

   1. 冗余行自动过滤：自动遍历表格行，定位第一行有效列数≥3、非全空的行作为待匹配表头行，自动过滤表头前的标题行、空行、备注说明行
   2. 合并单元格拆解：自动拆解所有跨行列合并单元格，将合并格原值完整填充到所有关联单元格，避免表头缺失
   3. 多行表头拼接：自动识别 2-3 行复合表头，将关联表头行文本拼接为完整表头名称（如「语文_卷面得分」），用于后续关键词匹配
   4. 无效列预过滤：自动过滤空值率≥50%、与成绩数据无关的纯文本无效列，仅保留符合数值特征、学生标识特征的列进入匹配环节

   

#### 1.1.2 分层权重关键词匹配核心规则（唯一提取模式）

-  内置教育成绩场景专属固定分层权重关键词体系，仅基于该体系完成字段匹配，不支持用户自定义修改，无任何其他匹配模式
-  采用「核心必选词 + 同义扩展词 + 强制排除词 + 数据校验规则」四维锁定匹配逻辑，彻底规避误匹配，具体规则如下：

表格

| 目标必选字段                                | 核心必选词（权重 100，命中即入围） | 同义扩展词（权重 60，补充匹配）        | 强制排除词（命中直接权重清零）           | 数据校验规则（必过，否则匹配失效）             |
| ------------------------------------------- | ---------------------------------- | -------------------------------------- | ---------------------------------------- | ---------------------------------------------- |
| 学生唯一标识                                | 姓名、学号                         | 学生姓名、考生号、准考证号             | 家长、班主任、老师、备注、联系人         | 列内文本占比≥90%、每行值唯一、空值率 < 10%     |
| 语文成绩                                    | 语文                               | 语文成绩、语文学科、语文得分、语文总评 | 语文老师、语文班主任、语文备注、语文排名 | 列内数值占比≥90%、数值范围 0-150、空值率 < 20% |
| 数学成绩                                    | 数学                               | 数学成绩、数学学科、数学得分、数学总评 | 数学老师、数学班主任、数学备注、数学排名 | 同语文成绩校验规则                             |
| 英语成绩                                    | 英语                               | 英语成绩、英语学科、英语得分、英语总评 | 英语老师、英语班主任、英语备注、英语排名 | 同语文成绩校验规则                             |
| 物理 / 化学 / 生物 / 历史 / 政治 / 地理成绩 | 对应科目全称                       | 对应科目 + 成绩 / 学科 / 得分 / 总评   | 对应科目 + 老师 / 班主任 / 备注 / 排名   | 同语文成绩校验规则                             |

-  匹配判定规则：

   1. 单列表头命中核心必选词且无排除词命中，同时通过数据校验规则，判定为匹配成功
   2. 单列表头仅命中同义扩展词，需额外通过数据校验规则，方可判定为匹配成功
   3. 必选字段要求：必须成功匹配「学生唯一标识」+ 至少 1 个科目成绩列，否则整体匹配失败

   

-  匹配失败处理：未满足必选字段匹配要求时，网页直接展示**数据匹配失败**全屏提示页，同步明确展示失败原因（如未识别到学生姓名字段、未识别到有效科目成绩列），无任何兜底处理、手动修正流程

#### 1.1.3 匹配成功后数据校验与分析入口

-  自动检测匹配成功的数据集内的缺失值、异常值（分数超满分、负数、非数值格式），在当前页下方生成极简校验提示，支持用户对当前数据进行临时剔除无效条目操作
-  提供两个独立的全屏分析入口按钮：「学生个体全科目分析」「班级单科目分析」，用户点击后进入对应全屏分析页，无任何其他导航干扰

### 1.2 学生个体全科目全屏分析页（单学生 + 全科目维度，网页原生渲染）

**页面布局**：全屏沉浸式布局，无顶部导航 / 侧边栏，仅在页面右上角保留「返回数据页」按钮，所有数据、表格、交互式图表均使用网页原生组件渲染（非图片），占满整个网页视口。

**分析范围**：基于当前匹配成功的数据集，针对用户选中的单个学生，对其本次数据内所有匹配成功的科目成绩进行全维度分析，全程仅聚焦当前单次数据，无任何历史对比。

#### 1.2.1 顶部核心指标卡片栏（横向铺满网页顶部）

-  横向排列 6-8 个核心指标卡片，占满网页顶部宽度，卡片内容实时响应学生切换：

   -  卡片 1：总分（大号字体展示）
   -  卡片 2：总得分率（带进度条可视化）
   -  卡片 3：总分数据集排名（标注「共 X 人参考」）
   -  卡片 4：优势科目数量（得分率≥85%）
   -  卡片 5：待提升科目数量（得分率 < 60%）
   -  卡片 6：科目均衡度（基于标准差计算，分「均衡 / 一般 / 不均衡」三级）

   

-  顶部左侧固定学生切换下拉框，支持用户快速切换查看不同学生的分析结果，切换后页面所有数据、表格、图表实时无刷新更新

#### 1.2.2 左侧：学生个体全科目成绩明细表（纵向占满网页左侧 50% 高度）

-  固定表头的交互式数据表格，纵向占满网页左侧区域，支持滚动查看所有科目，表格列固定包含：

   表格

   

   

   

   | 科目名称 | 本次得分 | 科目满分 | 得分率 | 超 / 低于平均分差值 | 科目排名 | 水平标注 |
   | -------- | -------- | -------- | ------ | ------------------- | -------- | -------- |
   |          |          |          |        |                     |          |          |

   

-  表格交互功能：

   -  支持按任意列点击排序
   -  水平标注自动着色：优势科目标绿、劣势科目标红、中等科目标灰
   -  鼠标悬停行高亮，同步联动右侧图表对应数据高亮

   

#### 1.2.3 右侧上半区：科目得分率横向柱状图（占满右侧上半区）

-  使用 ECharts/Chart.js 等网页原生图表库渲染的交互式横向柱状图，占满右侧上半区域：

   -  X 轴：得分率（0-100%），标注及格线（60%）、优秀线（85%）两条参考线
   -  Y 轴：所有科目名称
   -  交互功能：鼠标悬停显示具体得分、得分率、排名；点击科目可联动表格筛选对应行
   -  柱子自动着色：得分率≥85% 为绿色，60%-84% 为灰色，<60% 为红色

   

#### 1.2.4 右侧下半区：科目得分雷达图（占满右侧下半区）

-  使用网页原生图表库渲染的交互式雷达图，占满右侧下半区域：

   -  每个维度对应一个科目，径向轴为得分率（0-100%）
   -  雷达图背景标注及格线、优秀线两个同心圆参考线
   -  交互功能：鼠标悬停显示对应科目具体数据；支持切换「显示 / 隐藏数据集平均分雷达对比」

   

#### 1.2.5 底部：一键导出功能

-  页面底部固定悬浮导出栏，支持一键导出包含当前页面所有表格、图表的 PDF 格式分析报告，导出内容与网页展示完全一致

## 二、AI 纯文本辅助问答功能

### 2.1 纯文本题目输入能力

-  仅支持纯文本内容输入，提供基础文本编辑框，支持换行、分段，适配长题干、多小问题型的完整输入
-  内置 LaTeX 数学公式语法支持，满足数理化公式、特殊符号的精准输入，公式可实时渲染预览，确保题干输入准确性
-  支持用户手动标注题目科目、题型，辅助 AI 提升解答精准度，标注为非必填项，不影响基础问答功能使用
-  明确不支持任何图片、文件类题目上传，仅处理用户手动输入的纯文本内容

### 2.2 DeepSeek API 对接与核心解答能力

-  **API 自主配置**：支持用户自主填写 DeepSeek API 密钥，实时显示 API 调用状态、单次调用 token 消耗情况，调用配额完全由用户自有 API 密钥管控，平台无强制限制

-  **Prompt 优化适配**：预置分科目、分题型的标准化优化 Prompt 模板，自动适配输入题目的类型，提升解答准确性；同时支持用户自定义 Prompt 规则，自定义规则仅作用于当前会话

-  **多维度标准化解答输出**

   1. 核心答案区：单独展示题目的最终标准答案，支持一键复制
   2. 分步解题区：针对理科计算、逻辑推理类题目，输出完整递进的解题步骤，标注核心考点与得分关键点
   3. 知识点解析区：拆解题目涉及的核心知识点、公式定理、概念定义，补充基础原理讲解
   4. 双模式切换：支持用户一键切换「完整解答模式」/「仅思路引导模式」，引导模式下仅输出破题思路、易错点提醒，不直接给出最终答案，适配自主练习需求

   

### 2.3 多轮对话追问能力

-  支持用户针对当前题目的解答内容，进行上下文关联的纯文本追问，AI 基于完整对话上下文，输出针对性补充解答与拓展讲解
-  支持单会话内多题目连续提问，会话内对话记录仅临时缓存，页面刷新 / 会话结束后自动彻底清除，不进行任何持久化存储

### 2.4 解答反馈与临时优化

-  支持用户对本次解答结果进行星级评分，同时可选择「解答错误」「步骤缺失」「知识点讲解不清」等问题标签进行反馈
-  反馈后支持一键触发 AI 重新解答，平台基于用户反馈内容自动优化 Prompt，重新生成适配的解答结果
-  所有反馈数据仅临时作用于当前会话的重新解答，不进行云端存储、累计与模型训练
